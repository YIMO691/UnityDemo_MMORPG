## 今日主要变更
- 应用级管理
  - 引入 `GameManager`（单例 + DontDestroyOnLoad），负责核心管理器初始化、怪物配置加载入口、返回 Begin 流程统一入口（`ReturnToBeginFlow`）
  - 监听 `SceneManager.sceneLoaded`，确保切回 BeginScene 时自动打开 BeginPanel
- 场景装配两阶段
  - `GameSceneEntry` 拆分为 `TryAssembleScene(out controller)` 与 `CommitScene(controller)`，仅在装配成功后提交副作用（UI 打开、小地图绑定、玩家定位、导航代理接线、场景上下文写入）
  - 将小地图、运行时场景提交、导航代理接线抽出为独立组件（Assembler/Committer）
- 数据源与存档
  - 统一当前玩家数据访问入口为 `GamePlayerDataService`，移除对 `GameRuntime` 的业务写入
  - 存档槽位策略调整：`DataManager.GetNextAvailableSlotId` 改为尾部追加（返回 `MaxUsedSlotId + 1`），不在中间空槽插入
  - 创角流程更新：`CreateRoleFlowController` 写入当前玩家与槽位，再调用 `PlayerSaveService` 执行保存
- 资源与对象常量化
  - 集中资源路径：`AssetPaths`（DebugCanvas、PoolMonitorPanel、MapImageRoot、MonsterRoot、MonsterConfig 等）
  - 集中对象名：`ObjectNames`（MiniMapCamera、PlayerCameraRoot）
  - 集中导航常量：`NavigationConsts.PlayerAgentId`
  - MapPanel 加载图片改为通过 `AssetPaths.MapImageRoot + config.mapImage`
- Debug 能力
  - `GameSceneEntry.EnsureDebugCanvas`：加载 DebugCanvas 并挂接 `PoolMonitorPanel`（默认隐藏，开发/编辑器模式启用）
- UI 与流程修复
  - SettingPanel “返回标题”接入 `GameManager.ReturnToBeginFlow`，修复方法缺失右括号导致的编译错误
  - BeginPanel 返回时可见性恢复问题定位并通过 UI 层与路由统一入口修复
