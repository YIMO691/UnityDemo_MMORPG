# 二、完成内容

## 1. GameScene 初始化与进入流程

- 场景入口脚本：GameSceneEntry
  - InitScene：检查当前玩家数据 → CreateCamera → CreateAnimator → BindCamera → OpenMainPanel（见 [GameSceneEntry.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/GameScene/GameSceneEntry.cs)）。
  - CreateAnimator：实例化 PlayerArmature，校验 ThirdPersonController/CharacterController/PlayerCameraRoot/ModelRoot，并根据 classId 动态挂载职业模型（AttachRoleVisual）。
  - CreateCamera/BindCamera：实例化 MainCamera 与 PlayerFollowCamera，并将 CinemachineVirtualCamera 的 Follow/LookAt 绑定至角色控制器的 CinemachineCameraTarget。
  - 出生点：当 PlayerRuntimeData 含有效位置时从存档坐标生成，否则使用场景中配置的 playerSpawnPoint。
  - 打开 UI：进入场景后显示 MainPanel 作为主页面。

- 进入场景时机：
  - CreateRoleFlowController 在创角成功后，保存至槽位并加载 GameScene（见 [CreateRoleFlowController.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/Flow/CreateRoleFlowController.cs)）。
  - 运行期使用 GameRuntime.CurrentPlayerData 作为场景初始化的数据来源（见 [GameRuntime.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/Entity/Data/Runtime/GameRuntime.cs)）。

## 2. 运行时数据保存（位置与旋转）

- 数据结构：PlayerRuntimeData（见 [PlayerRuntimeData.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/Entity/Data/PlayerRuntimeData.cs)）
  - 保存当前 HP/MP/死亡状态、场景名、世界坐标 posX/posY/posZ、朝向 rotY、位置信息有效性标记 hasValidPosition。

- 保存服务：
  - RuntimeSaveService：将 Transform 写入 PlayerRuntimeData 并记录当前场景名（见 [RuntimeSaveService.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/Save/RuntimeSaveService.cs)）。
  - PlayerSaveService：封装保存流程，获取当前 PlayerData → 写入 Runtime 数据 → 设置回 DataManager → 根据当前槽位保存至磁盘（见 [PlayerSaveService.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/Save/PlayerSaveService.cs)）。
  - GameSceneEntry.SaveCurrentPlayerTransform：对外方法，供场景中合适时机（如交互点/定期/退出前）调用以落盘运行时数据。

- 多存档联动：
  - DataManager 扩展 currentSlotId、Set/Get/ClearCurrentSlotId，保存/读档时维持当前槽位（见 [DataManager.cs](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Framework/Managers/DataManager.cs)）。
  - SavePlayerDataToSlot/SaveCurrentPlayerDataToSlot 在写入时同步 currentSlotId，确保后续运行时保存无需额外指定槽位。

## 3. UI 与流程衔接

- 进入 GameScene 后自动打开 MainPanel（见 [GameSceneEntry.OpenMainPanel](file:///c:/Users/Administrator/Desktop/UnityDemo_MMORPG/Assets/Scripts/Game/GameScene/GameSceneEntry.cs#L260-L264)）。

  - GameRuntime.CurrentPlayerData 或 DataManager.Instance.GetCurrentPlayerData() 需在切场景前写入。

# 四、功能验证

- 创角 → 进场景：
  - 创角后自动加载 GameScene，场景中生成玩家与相机，主相机与跟随相机工作正常，打开 MainPanel 成功。

- 运行时保存与读档：
  - 在场景运行中调用 SaveCurrentPlayerTransform（或通过绑定事件触发）→ 退出 → 从相同槽位读取 → 再次进入 GameScene 出生点为存档位置与朝向。

- 常见问题：
  - 若角色控制器或相机缺失，GameSceneEntry 会打出错误日志并终止后续绑定。
  - 若 GameRuntime.CurrentPlayerData 为空，将提示数据缺失并跳过初始化。
