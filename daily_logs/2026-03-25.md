## 今日工程判断

今天已经不是单点修 bug，而是在把"角色控制"和"战斗主链路"往统一运行时架构上收。

最重要的变化不是新增某一个功能点，而是把多个此前分散的职责收敛成几条清晰主线：

| 主线 | 链路 |
|------|------|
| **控制主线** | Command → Brain → Executor → Navigator |
| **战斗主线** | Actor Capability → DamageRequest → BattleDamageService → DamageResult |
| **死亡主线** | DeathEvent → DeathRuntimeService → 清理/表现/销毁 |

这意味着项目从"原型功能堆叠"进一步转向"可扩展、可治理、可持续演进"的工程结构。
