# 2026-03-27 日报（玩家 HP/体力事件化 + 体力系统落地）

**今日目标**
- 解决“怪物打玩家但 UI 不更新”的断链问题
- 落地体力系统：只禁跑不禁走，兼容自动导航
- 补齐职业配置中的 maxStamina，保证存档/初始化数据合理

**关键变更**
- 玩家状态事件
  - 新增 PlayerHpChangedEvent：玩家扣血后广播 currentHp/maxHp
  - 新增 PlayerStaminaChangedEvent：体力变化时广播 currentStamina/maxStamina
- 主界面实时刷新
  - MainPanel 在 OnShow 订阅 HP/体力事件，在 OnHide 反订阅
  - 收到事件后更新 hpFill/staminaFill 的 fillAmount
- 体力系统（只锁 sprint）
  - 新增 PlayerStaminaSystem：
    - Sprint：每秒消耗体力
    - Walk：每秒恢复体力
    - Idle：恢复更快
    - 体力为 0：只禁跑，不禁移动
    - 恢复到阈值（30%）：重新允许跑步
  - 通过 PlayerLocomotionBrain.SetSprintEnabled(canSprint) 实现“禁跑不禁走”
- 自动导航兼容
  - PlayerNavigator：autoSprint 与 canSprint 叠加，体力不足时自动降为走路
- 数据与初始化
  - RoleClassConfig.json：为职业补齐 maxStamina 字段
  - PlayerEntity.Init：补齐/夹取 currentHp/currentStamina，并同步 isDead
  - PlayerCharacterAssembler：确保 PlayerStaminaSystem 在装配阶段挂载（避免漏挂导致规则不生效）

**验证清单**
- 怪物攻击玩家时，主界面血条实时下降
- Sprint 移动时体力下降；走路/Idle 时体力恢复
- 体力为 0 时自动降为走路（可继续移动）；恢复到 30% 后可再次跑
- 自动导航在体力不足时不再“硬跑”，不会卡住移动

**遗留与后续**
- HP/体力事件可以在 MainPanel 打开瞬间主动同步一次（避免 UI 初始值依赖事件）
- 玩家死亡后 UI/禁控/复活流程待接入（当前仅 isDead 标记）
