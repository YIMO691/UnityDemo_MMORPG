# 2026-03-26 日报（战斗→掉落→背包最小闭环打通）

**今日目标**
- 将“击杀→掉落→入包”的最小数据闭环落地，保持现有战斗主链不变
- 统一通过事件与服务接入：DeathEvent → LootRuntimeService → InventoryService
- 为后续 UI 和世界掉落物留出稳定接口

**关键变更**
- 战斗/死亡链路
  - BattleDamageService：击杀后发布 DeathEvent（既有）
  - 新：DeathRuntimeService 去重保护（防重复 SetDead），延迟销毁（保留死亡动画）
  - AppRuntimeInitializer：初始化顺序包含 DropTableConfigManager → LootRuntimeService → DeathRuntimeService
- 掉落体系
  - 新：DropTableConfig/DropEntryConfig/DropTableConfigList（最小数据结构）
  - 新：DropTableConfigManager（加载 Resources/Config/DropTableConfig.json）
  - 新：LootResolver（按权重抽一条，产出 itemId/count）
  - 新：LootRuntimeService（订阅 DeathEvent；HandleMonsterLoot→解析→日志/入包）
  - MonsterConfig：新增 dropTableId 字段；MonsterConfig.json 为测试怪挂载 dropTableId=1001
  - AssetPaths：新增 DropTableConfig 常量
- 背包与物品
  - PlayerData：新增 InventoryData（slotCount + slots）
  - GamePlayerDataService：EnsurePlayerDataSchema 统一补齐 inventoryData/slots/slotCount，旧档兼容；默认容量 20（最新为 50）
  - 新：ItemConfig/ItemConfigList、ItemConfigManager（加载 Resources/Config/ItemConfig.json）
  - 新：InventoryService.TryAddItem（叠堆→新开格，受 slotCount 限制）
  - AssetPaths：新增 ItemConfig 常量
  - 新配置：Resources/Config/ItemConfig.json、Resources/Config/DropTableConfig.json
- UI 事件与路由（不改主页面）
  - UIRouteNames：新增 InventoryPanel 路由名（弹窗）
  - 新：OpenInventoryPanelEvent（空载荷事件）
  - 新：InventoryUIController（订阅事件→ShowPanel<InventoryPanel>().Refresh()）
  - GameManager：InitCoreManagers 初始化 InventoryUIController

**验证清单**
